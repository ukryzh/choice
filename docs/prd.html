Product Requirements Document

1. Overview

1.1 Purpose

This document defines the product requirements for a mobile menstrual cycle tracking application with a strong focus on data privacy and secure data portability between user devices.

The PRD defines:

  • User journeys
  • Functional requirements
  • UX and privacy principles
  • MVP readiness criteria

1.2 Product Vision

To create a privacy-first menstrual cycle tracking application in which users have full control over their data — including storage, transfer, and deletion — while the developer has no access to the data content.

1.3 In Scope (MVP)

  • Menstrual cycle tracking
  • Cycle history and predictions
  • Symptom logging
  • Local data storage
  • Optional encrypted cloud backups
  • Data recovery on a new device

1.4 Out of Scope

  • Monetization
  • Analytics and tracking
  • Medical recommendations
  • Integrations with clinics or healthcare providers

2. User Personas

Persona 1: Privacy-Conscious User

  • Does not trust mass-market health applications
  • Is concerned about personal data leaks
  • Prefers a simple and transparent interface

Persona 2: Device-Switching User

  • Frequently changes devices
  • Fears losing personal data
  • Is willing to use cloud storage if strong encryption is guaranteed

3. Design & Privacy Principles

3.1 Privacy First

  • All data belongs exclusively to the user
  • All data is encrypted before being sent to the server
  • The server has no access to decrypted user data

3.2 User Control

  • Cloud storage is strictly opt-in
  • Users can delete their data at any time
  • The application works without registration

3.3 Simplicity

  • Minimal number of screens
  • Clear and understandable language
  • No overloaded or complex settings

4. User Journeys

4.1 First App Launch

Description:
The user opens the application for the first time.

Flow:

  1. Welcome screen
  2. Short explanation of privacy principles
  3. Mode selection:
    • Sign in and synchronize
    • Skip authorization
  4. Set a four-digit app access code
  5. Navigate to the main screen (Calendar)

Acceptance Criteria:

  • The app can be used without account creation
  • Cloud usage is optional

4.2 Adding a Cycle

User Story:
As a user, I want to add the start date of my cycle so that I can track regularity.

Flow:

  1. Tap “Log period”
  2. Select dates
  3. Select symptoms (optional)
  4. Save

Edge Cases:

  • Overlapping dates
  • Missing cycle end date

Acceptance Criteria:

  • Data is saved
  • History is updated
  • Predictions are recalculated

4.3 Viewing History and Predictions

User Story:
As a user, I want to see past cycles and upcoming predictions to plan my activity.

Flow:

  1. Open history screen
  2. View calendar
  3. View predicted cycle dates

Acceptance Criteria:

  • Predictions update automatically

4.4 Symptom Logging

User Story:
As a user, I want to log symptoms for a specific day of my cycle.

Flow:

  1. Select a day
  2. Select symptoms
  3. Save

Acceptance Criteria:

  • Symptoms are linked to a specific date
  • Symptoms can be edited

4.5 Enabling Cloud Backup

User Story:
As a user, I want to enable encrypted cloud backups to avoid losing my data.

Flow:

  1. Open Profile screen
  2. Explanation of:
    • what data is sent
    • how encryption works
    • why the server has no access
  3. User confirmation

Acceptance Criteria:

  • Data is encrypted on the client
  • Data transfer happens only after explicit consent

4.6 Data Recovery on a New Device

User Story:
As a user, I want to restore my data on a new device so I can continue using the app.

Flow:

  1. Install the application
  2. Select “Restore data”
  3. Authenticate / confirm access
  4. Download and decrypt data

Acceptance Criteria:

  • All data is fully restored
  • No developer access to user data

4.7 Data Deletion

User Story:
As a user, I want to delete all my data to ensure it cannot be accessed by third parties.

Flow:

  1. Open Profile screen
  2. Select “Delete all data”
  3. Choose data type to delete (local / cloud / all)
  4. Confirm action

Acceptance Criteria:

  • Local data is deleted
  • Cloud backups are deleted

5. Functional Requirements (Summary)

  • Local data storage
  • Optional cloud backups
  • End-to-end encryption
  • View, edit, and delete data
  • Offline support

6. Non-Functional Requirements

Performance

All user actions must complete within ≤ 1 second.

Security

  • Client-side encryption
  • No server access to user data

Reliability

  • No data loss when switching devices

Accessibility

  • Readable fonts

7. Compliance

  • GDPR compliance
  • Privacy by Design / Privacy by Default
  • Google Play policy compliance

8. Success Metrics (MVP)

  • Application published on Google Play
  • All user journeys work without errors
  • No analytics or tracking tools
  • Users can successfully restore data
